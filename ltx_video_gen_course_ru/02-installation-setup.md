# 02 — Установка и настройка

Этот модуль приведёт вас к работающему ComfyUI с LTX-2.3, прогоняющим
пример воркфлоу целиком. Не пропускайте smoke-тест в конце.

> **Помните про энкодер.** LTX-2.3 использует **мультимодальный энкодер
> на базе Gemma** (текст + аудио), а не T5 / CLIP. Это значит, что три
> отдельные больших файла весов должны лежать в правильных папках,
> прежде чем что-либо загрузится: чекпоинт LTX-2.3, энкодер Gemma и VAE
> LTX-2.3 (видео + аудио). Если вы следуете старому туториалу по
> LTXV 0.9.x, показанный там шаг «CLIPLoader / T5» здесь неприменим.

## Проверка реалистичности железа

LTX-2.3 тяжелее, чем LTXV 0.9.x, потому что энкодер Gemma большой, а
ко-генерация аудио добавляет латентные каналы. Воспринимайте таблицу
ниже скорее как ориентир, чем как истину в последней инстанции —
сверяйтесь с официальной карточкой модели LTX-2.3 для актуальных цифр.

| VRAM | Что реально с LTX-2.3 |
|------|-------------------|
| 8 ГБ | Очень впритык. OOM без агрессивного оффлоадинга (fp8 + CPU-offload Gemma). Для обучения не рекомендуется. |
| 12 ГБ | Низкое разрешение, короткие клипы, веса в fp8 и Gemma в оффлоаде. Пригодно для обучения. |
| 16 ГБ | Среднее разрешение, клипы 4–5 с с аудио, fp8. Приемлемо. |
| 24 ГБ (3090/4090) | Нативное разрешение, клипы 5–10 с с аудио, стеки из нескольких LoRA. Оптимально. |
| 48 ГБ+ | Полное разрешение, длиннее клипы, тяжёлый стекинг LoRA, батчинг, Gemma в высокой точности. |

CPU и RAM значат больше, чем в LTXV 0.9.x — оффлоадинг Gemma уходит в
системную RAM, так что 32 ГБ+ RAM — реальное преимущество. NVMe SSD
сильно помогает, потому что веса моделей большие, а веса Gemma в
частности подгружаются/выгружаются во время мульти-LoRA-воркфлоу.

## Устанавливаем ComfyUI

Следуйте официальному пути установки для вашей ОС. Канонический источник
— GitHub-репозиторий `comfyanonymous/ComfyUI` — прочитайте его README и
возвращайтесь. Минимальный контур:

```
git clone https://github.com/comfyanonymous/ComfyUI
cd ComfyUI
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
# Установите PyTorch с версией CUDA, подходящей под ваш драйвер — следуйте
# селектору на pytorch.org, не копируйте вслепую версию из туториала.
python main.py
```

Откройте `http://127.0.0.1:8188` в браузере. Вы должны увидеть пустой
нод-канвас с воркфлоу по умолчанию.

## Устанавливаем ComfyUI-Manager

ComfyUI-Manager не опционален для этого курса. Он занимается установкой
кастомных нод, их обновлением и разрешением ошибок о недостающих нодах,
когда вы загружаете чужой воркфлоу.

```
cd ComfyUI/custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager
```

Перезапустите ComfyUI. Вы увидите кнопку «Manager» в верхнем меню.

## Устанавливаем кастомные ноды LTX-2.3

LTX-2.3 поставляется со своим нод-паком для ComfyUI. Через Manager:

1. Откройте Manager -> Install Custom Nodes.
2. Ищите «LTX-2» или «LTX Video» (официальный пак от Lightricks).
   Выбирайте пак LTX-2.3, а не старый LTXV 0.9.x, если перечислены оба.
3. Установите и перезапустите ComfyUI.

Как альтернатива, через git (проверьте текущий канонический репо —
Lightricks публикует ноды LTX-2 либо в продолжении `ComfyUI-LTXVideo`,
либо в отдельном LTX-2 репо):

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/Lightricks/ComfyUI-LTXVideo
```

Проверка: после перезапуска двойной клик по канвасу и поиск `LTX` —
вы должны увидеть специфичные для LTX-2.3 ноды loader, encoder, sampler и
VAE decode. Точные имена нод зависят от версии пака; используйте *те
имена, которые видите* в своей установке, когда перестраиваете графы в
этом курсе. Признак того, что у вас LTX-2.3 (а не 0.9.x) — наличие ноды
**Gemma encoder** в официальном примере воркфлоу и **аудио-выходного**
пина на семплере или декодере.

## Скачиваем веса модели

Для LTX-2.3 нужно минимум **четыре** группы файлов (vs. три для старого
LTXV). Кладите их по путям ниже (относительно `ComfyUI/`):

| Что | Куда | Замечания |
|------|-------|-------|
| Основной чекпоинт LTX-2.3 | `models/checkpoints/` (или LTX-специфичная подпапка, которую ожидает нод-пак) | Веса видео+аудио DiT. Варианты точности: fp16 / bf16 / fp8. fp8 на 12–16 ГБ. |
| Энкодер Gemma для LTX-2.3 | `models/text_encoders/` (иногда выделенная подпапка вроде `models/LLM/` или `models/gemma/` — следуйте указаниям нод-пака) | Мультимодальный энкодер. Большой (несколько ГБ). **Не подменяйте T5** — LTX-2.3 не загрузится. |
| LTX-2.3 VAE | `models/vae/` | Видео-VAE. Часто поставляется вместе с аудио-VAE или как отдельный файл. |
| LTX-2.3 аудио-VAE (если отдельно) | `models/vae/` или `models/audio_vae/` | Декодирует аудио-латенты в waveform. Некоторые релизы встраивают его в основной файл VAE. |

Источник: Hugging Face-репозиторий Lightricks для LTX-2.3. Читайте
карточку модели в день скачивания — это источник истины о том, какие
файлы объединены, а какие отдельны в скачиваемом релизе.

**Выбор точности:**

- **bf16** или **fp16** на 24 ГБ+. Максимальное качество.
- **fp8** на 12–16 ГБ. Небольшая потеря качества, большая экономия VRAM.
- Энкодер Gemma обычно можно загружать в меньшей точности (int8 / fp8)
  независимо от основного DiT. Если ваш нод-пак это поддерживает,
  пользуйтесь — Gemma большая и как энкодер терпима к точности.

## Рекомендуемые вспомогательные кастомные ноды

Устанавливайте через Manager. Все пригодятся в дальнейших модулях:

- **ComfyUI-VideoHelperSuite** — `VHS_VideoCombine`, `VHS_LoadVideo` и т. д.
  Необходимо для сохранения выходов как mp4/webm и загрузки референсных
  клипов.
- **ComfyUI-Frame-Interpolation** — ноды RIFE / FILM для удвоения частоты
  кадров на пост-обработке.
- **ComfyUI-Custom-Scripts** (pythongosssss) — quality-of-life: автодополнение
  нод, история промптов.
- **rgthree-comfy** — power-инструменты: muters, bypassers, context-ноды.
  Делает сложные графы управляемыми.
- **ComfyUI-Impact-Pack** — условное семплирование, фильтры изображений.
  Полезно для I2V-препроцессинга.

## Структура папок, которой вы будете пользоваться

Точные имена файлов берутся из скачанного вами релиза LTX-2.3 —
копируйте имена так, как они есть. Важна именно *форма*:

```text
ComfyUI/
  models/
    checkpoints/
      ltx-2.3-fp8.safetensors          # или ltx-2.3-bf16.safetensors
    text_encoders/                     # или models/LLM/, по нод-паку
      ltx-2.3-gemma-encoder.safetensors
    vae/
      ltx-2.3-vae.safetensors          # видео-VAE (+ аудио, если встроено)
      ltx-2.3-audio-vae.safetensors    # только если распространяется отдельно
    loras/
      style/                           # организуйте по назначению
        90s-spiderman-v2.safetensors
        anime-madhouse-90s.safetensors
      character/
      motion/
  user/default/workflows/
    01-t2v-basic.json
    02-i2v-basic.json
    ...
  output/
    2026-04-20-spiderman-test-01/
      0001.mp4                         # по умолчанию с синхронным аудио
      prompt.txt
```

Дисциплина с подпапкой по дате для каждого вывода важна. Видео-итерации
быстро генерируют мусор; вам нужно знать, какой mp4 соответствовал
какому промпту спустя две недели. С LTX-2.3 вы ещё и сохраняете mp4 с
аудио — держите аудио при просмотре; убирайте на этапе сдачи, если
предпочитаете писать звук на пост-продакшне.

## Smoke-тест

Прежде чем идти дальше, прогоните **официальный пример воркфлоу
LTX-2.3**. Это самый важный шаг настройки — это ваша референс-истина по
именам нод, разводке пинов и конфигурации энкодера. Каждый воркфлоу
далее в этом курсе — это модификация этого примера.

1. В ComfyUI: меню Workflow -> Browse Templates -> найти примеры LTX-2 /
   LTX-2.3 (устанавливаются с нод-паком). Или загрузите пример `.json`
   из папки `example_workflows/` нод-пака.
2. Используйте дефолтный промпт. Больше ничего не меняйте.
3. Поставьте в очередь. Смотрите на консоль на предмет ошибок.
4. Ожидается: mp4 длиной 2–5 секунд, сохранённый в `output/`, **с
   синхронным аудио**. Если у вас отключён звук, воспроизведите со
   звуком — аудио здесь это фича, а не баг.

Если это работает — вы готовы. Если нет, 95% отказов попадают в:

- **Отсутствует файл модели** — в ноде-лоадере выпадающий список пуст или
  красный. Фикс: реально скачайте файл в нужную папку, потом нажмите
  кнопку обновления в правом верхнем углу ComfyUI.
- **Отсутствуют ноды (красные ноды)** — не установлен нод-пак. Фикс:
  Manager -> Install Missing Custom Nodes.
- **OOM** — уменьшите разрешение, уменьшите количество кадров,
  переключитесь на fp8 веса, включите CPU-offload для Gemma, включите
  `--lowvram` при старте.
- **Энкодер Gemma не найден** — либо не скачали, либо положили не в ту
  подпапку (`text_encoders/` vs. `LLM/` vs. `gemma/` варьируется по
  паку), либо файл обрезан. Перечитайте README нод-пака по точной папке.
- **«Ожидался T5» или красная нода `CLIPLoader`** — вы случайно
  загружаете **старый пример воркфлоу LTXV 0.9.x** вместо LTX-2.3.
  Откройте правильный пример LTX-2.3 и выбросьте старый.
- **В mp4-выходе нет аудио** — не загружен аудио-VAE, или у ноды
  `VHS_VideoCombine` не прокинут аудио-входной пин. Осмотрите
  аудио-путь в примере воркфлоу и повторите его.

Когда smoke-тест даёт воспроизводимый mp4 с аудио, переходите к
[03-ltx-model-fundamentals.md](03-ltx-model-fundamentals.md).
