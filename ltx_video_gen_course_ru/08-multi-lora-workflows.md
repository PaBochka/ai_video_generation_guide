# 08 — Воркфлоу с несколькими LoRA

Одиночные LoRA — это легко. Стеки — вот где ремесло. Этот модуль о том,
как использовать 2–5 LoRA вместе так, чтобы они не дрались.

## Зачем стекать

Типичные полезные стеки:

- **Style + Character.** 90s-anime-style-LoRA + character-LoRA вашего
  героя. Стиль из одной, идентичность из другой.
- **Style + Environment.** Вид киноплёнки + конкретный сеттинг
  (cyberpunk city, medieval village).
- **Style + Motion.** Визуальный стиль + motion-LoRA, смещающая в
  конкретные движения камеры или типы действий.
- **Style + Style (блендинг).** Два стиля на половинной силе каждый,
  чтобы получить интерполированный лук. Тонко; работает иногда.

## Базовый стекинговый граф

```
   Checkpoint
        |
   LoraLoader(style)        strength_model 0.8 / strength_clip 0.8
        |
   LoraLoader(character)    strength_model 0.7 / strength_clip 0.6
        |
   LoraLoader(motion)       strength_model 0.5 / strength_clip 0.3
        |
   Sampler
```

Порядок значит меньше, чем утверждают форумы — математика это
аддитивные low-rank дельты — но организация по «от общего к частному»
(style -> character -> shot-specific) делает граф читаемее.

## Разрешение конфликтов

Когда две LoRA пересекаются в том, что меняют, они конкурируют.
Симптомы:

- Стиль деградирует при добавлении character-LoRA.
- Идентичность персонажа деградирует при увеличении силы стилевой
  LoRA.
- Вывод становится мутным, пересвеченным или странно пластиковым —
  иногда всё три одновременно.

Playbook:

1. **Отключите все LoRA, кроме первичной для шота.** Вывод выглядит
   правильно? Если нет, проблема в первичной; чините её до стекинга.
2. **Добавляйте LoRA по одной.** После каждого добавления проверяйте
   вывод.
3. **Если новая LoRA ломает, снизьте её силу на 0.2 и повторите.**
   Продолжайте, пока она не будет вкладывать, не ломая.
4. **Если две LoRA фундаментально несогласуемы** (например, два стиля
   с очень разными центрами), признайте, что стек неправильный.
   Выберите одну.

Ноды bypass/mute из `rgthree-comfy` ускоряют это — можно кликом
включать/выключать LoRA без перепроводки.

## Паттерн «якорь + акцент»

Самая чистая ментальная модель для стеков 2–3:

- **Anchor LoRA** (0.8–1.0) — доминирующий лук. Обычно стиль.
- **Accent LoRA(s)** (0.3–0.6) — толчки. Character, motion,
  environment.

Не запускайте два якоря. Если хотите два сильных стиля, блендите
через композицию в image-space (region-prompting, img2img-проходы),
а не через двойную перекрутку LoRA.

## Region-scoped LoRA (T2I-стадия)

На T2I-стадии — где вы делаете ключевые стилы — региональные LoRA
возможны через нод-паки вроде регионального кондиционирования
ComfyUI. Вы говорите графу:

- Применять character-LoRA только к левой половине кадра (где стоит
  персонаж).
- Применять environment-LoRA только к маске фона.

Это зрелое для SDXL/Flux и менее — для LTX-2.3. На практике
используйте региональные LoRA на T2I-стадии для чистых ключевых
стилов, затем I2V'ите стил в LTX-2.3 без региональных трюков.

## Time-scoped LoRA (видео-стадия, экспериментально)

Некоторые коммьюнити-воркфлоу поддерживают расписание силы LoRA во
времени (кадрах). Пример: сила нарастает с 0.0 на кадре 0 до 0.8 на
кадре 50 для эффекта «стиль проявляется». На LTX-2.3 считайте это
экспериментальным — полезно для переходов, не для продакшн-шотов.

Если хотите попробовать: ищите ноды в стиле `LoraScheduler` со
входами временного диапазона или специфичные для LTX-2.3 расширения
LoRA-планирования в коммьюнити-пакетах кастомных нод. Имя ноды может
варьироваться — паттерн «LoRA с входом start/end frame».

## Блендинг двух стилей

Воркфлоу:

```
   Checkpoint
        |
   LoraLoader(style_A)   strength 0.5 / 0.5
        |
   LoraLoader(style_B)   strength 0.5 / 0.5
        |
   Sampler
```

Правила большого пальца:

- Оба на 0.5/0.5 — стартовая точка. Хорошие пары работают;
  большинство нет.
- Если A доминирует, опустите A до 0.4 и поднимите B до 0.6.
  Сохраняйте суммарный «бюджет» около 1.0.
- Генерируйте сетку 3×3 с силами (0.3, 0.5, 0.7) по каждой оси.
  Оценивайте на глаз. Выбирайте sweet spot.
- Триггеры: включайте оба набора. Некоторые блендинги оживают
  только когда присутствуют оба триггера.

## Примеры стеков для этого курса

### Дорожка A — SpiderMan 90-х (T2I-стадия ключевого стила на SDXL или Flux)

```
LoRA                              model / clip
--------------------------------------------
90s-american-animation-style      0.85 / 0.80   (якорь)
spider-man-1994-character         0.65 / 0.55   (акцент — только если есть отдельный)
comic-book-inking                 0.35 / 0.30   (акцент — усиливает контуры)
```

Триггеры: в начало style-триггер + character-триггер через запятую.

### Дорожка B — Аниме (T2I-стадия ключевого стила)

```
LoRA                              model / clip
--------------------------------------------
madhouse-90s-style                0.80 / 0.70   (якорь)
character-name                    0.70 / 0.60   (акцент)
cel-shading-painterly             0.30 / 0.25   (акцент)
```

### Дорожка C — Микс-медиа

Обычно одна «film look» LoRA на лайв-экшн-плейты + отдельная
стилизованная LoRA на анимационные оверлеи. Делается двумя
отдельными генерациями, не стекингом.

### Сторона LTX-2.3 — когда LoRA существуют

На момент написания совместимые с LTX-2.3 LoRA существуют, но их
меньше. Реалистичный стек стадии LTX-2.3:

```
LoRA                              model / clip
--------------------------------------------
ltxv-style-cel-animation          0.80 / 0.80   (если доступно под ваш стиль)
ltxv-motion-action                0.50 / 0.40   (если доступно)
```

Часто LTX-2.3 запускают **без LoRA** и полагаются на I2V-входное
изображение, несущее стиль — и оно неплохо несёт, первые 2–3 секунды.
Дрейф после 3 с — сигнал либо укоротить клип, либо найти стилевую
LoRA на стороне LTX-2.3.

## Таблица отладки мульти-LoRA

| Симптом | Что попробовать первым |
|---|---|
| Черты персонажа «протекают» в стиль (все выглядят как обученный перс) | Снизьте clip-силу character-LoRA; сдвиньте триггер персонажа позже в промпте |
| Стилевая LoRA подавляет персонажа | Снизьте силу модели стиля на 0.1; поднимите силу модели персонажа на 0.1 |
| Все цвета не те | Одна LoRA обучалась на сильно фильтрованных данных; снизьте её силу |
| Вывод пересвечен и «хрустит» | Общий «бюджет» LoRA > 1.5; ребалансируйте |
| LoRA будто не имеет эффекта | Несовместимость базовой модели, или отсутствие триггера, или отключена bypass-нодой |

## Упражнение — Постройте свой стек

Выберите свою дорожку. На T2I-стороне:

1. Протестируйте каждую LoRA по отдельности на силе 0.8 с нейтральным
   промптом. Запишите, что делает.
2. Стекайте их в порядке «якорь + акцент». Начните с якоря на 0.8,
   акцентов на 0.4.
3. Сгенерируйте 9 изображений своим проектным промптом. Оцените.
4. Подкрутите силы. Перегенерируйте 9. Зафиксируйте, когда довольны.
5. Сохраните это как T2I-воркфлоу `05-keystill-track[A|B|C].json`.

На стороне LTX-2.3: загрузите T2I-выход как I2V-вход, прогоните
keyframe-воркфлоу из модуля 06. Если доступна и совместима стилевая
LoRA LTX-2.3, добавьте её на 0.8 и сравните.

## Бюджет LoRA — ментальная модель

Думайте о силе LoRA как о бюджете. Каждая LoRA тратит часть; общая
трата выше ~1.5 склонна к перегоранию вывода. Примерная бухгалтерия:

```text
style_anchor    strength 0.85  ->  0.85 budget
character       strength 0.65  ->  0.65 budget
detail_accent   strength 0.30  ->  0.30 budget
------------------------------
total spent                        1.80  (опасная зона)
```

Бюджеты — эвристика: две LoRA, меняющие непересекающиеся аспекты
(одна трогает цвет, другая позу), могут сосуществовать выше 1.5
без проблем. Две LoRA, трогающие одно и то же (два стилевых якоря),
взрываются на сумме 1.2. Ведите учёт неформально; когда вывод
деградирует, первое, что надо снизить, — общая трата.

### Рецепты бюджета

```text
Консервативно (безопасно, минимум конфликта):
  якорь 0.70 + акцент 0.35              всего 1.05

Стандарт (дефолт для обучения):
  якорь 0.80 + акцент 0.45 + 0.30       всего 1.55

Агрессивно (только с очень совместимыми LoRA):
  якорь 0.95 + акцент 0.60 + 0.40       всего 1.95

Блендинг (два якоря):
  anchor_A 0.55 + anchor_B 0.50         всего 1.05
```

Если не знаете, где вы, начните со Стандарта. Сдвигайтесь, когда
увидите, где узкое место.

## Рецепты стеков — копируйте и подгоняйте

Конкретные условно-хорошие паттерны. Все T2I-стадийные, если не
сказано иное. Подменяйте имена LoRA на доступные под вашу базовую
модель.

### Рецепт S1 — «Hero-шот 90s action cartoon»

Для ключевых стилов дорожки A.

```text
Base: SDXL или Flux.1
Lora 1: 90s-saturday-morning-animation       0.85 / 0.80   (якорь)
Lora 2: comic-book-inking                    0.40 / 0.30   (акцент: толщина линий)
Lora 3: cel-shading-two-tone                 0.25 / 0.20   (акцент: форма теней)
Триггеры (начало промпта): [style_trigger_1], [style_trigger_2],
Эффективная сумма: ~1.50
```

Следите: если цвета «хрустят», сначала снизьте акцент толщины линий;
если линии становятся размытыми, поднимите его.

### Рецепт S2 — «1990s Madhouse anime drama»

```text
Base: SDXL или Flux.1
Lora 1: madhouse-90s-style                   0.80 / 0.75   (якорь)
Lora 2: painterly-background                 0.35 / 0.25   (акцент: только фон)
Lora 3: character_lora (если свой)           0.70 / 0.55   (акцент: идентичность)
Триггеры: [style_trigger], [char_trigger],
Эффективный итог: ~1.85 (растянуто — следите за перегоранием)
```

Если идентичность персонажа деградирует на этой сумме: снизьте силу
модели `painterly-background` до 0.25. Это самый «одноразовый» нод.

### Рецепт S3 — «Ghibli-esque акварель»

```text
Base: SDXL
Lora 1: ghibli-inspired-watercolour          0.85 / 0.75
Lora 2: soft-natural-lighting                0.35 / 0.30
Итог: ~1.20
```

Обычно без character-LoRA; лица персонажей Ghibli-стиля чувствительны,
и большинство character-LoRA взрывают лук. Описывайте персонажа
промптом.

### Рецепт S4 — «Стилизованный ротоскоп (микс-медиа)»

```text
Base: SDXL
Lora 1: sharpie-rotoscope-ink                0.90 / 0.80
Lora 2: high-contrast-film-still             0.40 / 0.35
Итог: ~1.30
```

Рассчитано применять к *реальному фото-img2img-входу* на умеренном
denoise (0.55–0.70). Акцент высокого контраста помогает ротоскопу
«закусывать» фотографический свет.

### Рецепт S5 — «Comic book splash page (статичный)»

```text
Base: SDXL
Lora 1: silver-age-comic-art                 0.75 / 0.70
Lora 2: halftone-dots                        0.25 / 0.20
Lora 3: dynamic-pose                         0.40 / 0.30
Итог: ~1.40
```

Даёт очень сильные композиции, но ужасную временную консистентность
при подаче в I2V (halftone-точки мерцают). Хорошо для одноразовой
T2I-стадии; планируйте под это мерцание дальше.

### Рецепт S6 — «Блендинг двух стилей, painterly realism»

```text
Base: Flux.1
Lora 1: oil-painting-impasto                 0.55 / 0.45
Lora 2: cinematic-photograph                 0.50 / 0.45
Итог: ~1.05
```

Оба примерно на половинной силе. Даёт лук «фотография масляной
картины». Чувствительно к промпту; слова субъекта дрейфуют к стилю,
у которого больше обучающих прецедентов для данного субъекта.
Подстраивайте соотношение по шоту.

## Разрешение конфликтов — три разобранных кейса

### Кейс 1 — Character-LoRA перебивает стилевую

**Симптом:** character-LoRA на 0.7 превращает всё в
фотореалистичные обучающие снапшоты, стирая аниме-стиль.

**Диагноз:** character-LoRA обучалась на фотореалистичных данных,
так что её «идентичность» включает фотографический рендер-стиль.

**Что пробовали (и почему):**
- Снизить силу модели character до 0.5. Идентичность падает, стиль
  возвращается. Приемлемо, если идентичность ещё читается.
- Оставить силу модели character на 0.7, но поставить clip-силу на
  0.2. Идентичность держится через UNet, текстовые промпты сохраняют
  стилевое влияние.
- Перегенерировать T2I-референс персонажа *сначала с применённой
  аниме-стилевой LoRA*, затем переобучить новую character-LoRA на
  этих стилизованных изображениях. Дорого; для долгого проекта —
  стоит.

### Кейс 2 — Две стилевые LoRA выдают кашу

**Симптом:** сочетание «cel-shaded» + «watercolour» на 0.5 каждая
не даёт ни того, ни другого — неуютная середина с плоским цветом,
но мягкими краями. Вывод читается как «AI slop».

**Диагноз:** два стиля фундаментально противопоставлены (жёсткие vs.
мягкие края, плоские заливки vs. растекания). Нет связной середины,
через которую модель могла бы интерполировать.

**Что пробовали:**
- Выбрать один. Всегда правильный ответ для противоположных стилей.
- Если *намерение* было в конкретно блендовом луке (например, «cel-
  персонаж на акварельном фоне»), не блендируйте LoRA-стеком —
  используйте **региональное кондиционирование** (маска на LoRA, см.
  ниже) или генерируйте двумя проходами и композьте.

### Кейс 3 — Акцент-LoRA будто ничего не делает

**Симптом:** добавили «detail-accent» LoRA на 0.4; вывод выглядит
идентично без неё.

**Диагноз:** три возможности, проверьте по порядку.

1. Нет триггера — добавьте.
2. Несовпадение базовой модели — обучалось на SDXL 1.0, вы на SDXL
   Turbo. Мелко, но достаточно, чтобы адаптер промахивался по
   весам.
3. LoRA реально слабая и нужна 1.0+, чтобы проявиться.

**Фикс:** добавьте триггер, бустните до 1.0, потом 1.3. Если по-
прежнему ничего, замените LoRA — она плохо обучена.

## Региональная (пространственная) мульти-LoRA — полный пример

Это эпоха SDXL/Flux; поддержка регионального кондиционирования в
LTX-2.3 незрелая на момент написания. Паттерн через нод-пак
регионального кондиционирования:

### Сетап

Цель: персонаж на левой половине, писаный фон на правой, у каждого
свой LoRA-стек.

### Нод-граф

```text
                 [Base Checkpoint]
                        |
             LoraLoader(char_anchor)  strength 0.8
                        |
             LoraLoader(bg_anchor)    strength 0.7
                        |
                        +----> model out
                        |
   [Mask: left-half]    [Mask: right-half]
          |                    |
   ConditionSetArea (промпт персонажа + триггер char-LoRA, маска: left)
          |
   ConditionSetArea (промпт фона + триггер bg-LoRA, маска: right)
          |
   ConditioningCombine -----> Sampler
```

LoRA загружены в UNet глобально, но каждый регион получает свой
*промпт* и поэтому активирует только концепты релевантной LoRA
через триггерные слова в этом регионе.

### Подвохи

- Персонажи на границе маски могут выглядеть «разрезанными».
  Размывайте края маски на 40–80 пикселей.
- Обе LoRA применяются глобально, так что ждите протекания —
  левый фон частично унаследует смещение bg-LoRA. Обычно это норм
  и визуально объединяет.
- Не ставьте больше 2–3 регионов. Каждый регион умножает промпт-
  сложность для модели.

## Time-scoped LoRA — конкретный паттерн

На видео-стадии некоторые нод-паки сообщества поддерживают
покадровое расписание силы. Паттерн для «стиль проявляется за 2
секунды»:

```text
Frame 0       -> LoRA strength 0.0
Frame 24      -> LoRA strength 0.4
Frame 48      -> LoRA strength 0.8  (прибыл)
Frame 48+     -> LoRA strength 0.8  (удержание)
```

Кейсы:

- **Style-reveal.** Переход в сне-сцену, где реальность
  фотореалистична, а сны — акварель.
- **Сцены трансформации.** Персонаж морфит стиль во время «power
  up».
- **Эффекты линзы/камеры, привязанные к сюжетным битам.** Нарастите
  LoRA «дисторсии», когда персонаж теряет хватку с реальностью.

Экспериментальная территория на LTX-2.3. Если нативно не
поддерживается, обходите хаком: генерируйте два клипа (A: без LoRA,
B: LoRA на 0.8) и crossfade'ьте в редакторе. Дешевле, 80% того же.

## Power-user-паттерны с rgthree-comfy

`rgthree-comfy` добавляет quality-of-life ноды, делающие мульти-LoRA
графы сопровождаемыми. Основные паттерны:

### Паттерн 1 — Mute/bypass на каждую LoRA

Замените цепочку `LoraLoader` на `rgthree Power Lora Loader` (или
аналог). Каждая LoRA получает on/off-тумблер и слайдер силы внутри
ноды. A/B-тестирование становится кликом вместо перепроводки.

### Паттерн 2 — Context bus

Прокиньте ноды `Context`, несущие `model`, `clip`, `vae`, `positive`,
`negative`, `latent` через граф одной шиной. Канвас перестаёт быть
спагетти. Нижестоящие ноды цепляются к шине, а не получают пять
отдельных проводов.

### Паттерн 3 — Реруты и группы

Раскрашивайте группы: «Loaders» (чекпоинт + LoRA), «Conditioning»
(промпты), «Sampling» (семплер + VAE-декод), «Output» (save +
video combine). Сворачивайте те группы, которые не редактируете.

### Паттерн 4 — Seed-ноды

Выделенные `Seed`-ноды подают сид в семплер. Можно залочить/
разлочить один seed-нод на весь граф, и сид отображается в заголовке
ноды. Обязательно при сцепке шотов с общим сидом.

## Упражнение A — Постройте стек

(Как раньше, без изменений.) Выберите дорожку. На T2I-стороне:

1. Протестируйте каждую LoRA по отдельности на 0.8 с нейтральным
   промптом. Запишите, что делает.
2. Стекайте в порядке «якорь + акцент». Начните с якоря на 0.8,
   акцентов на 0.4.
3. Сгенерируйте 9 изображений своим проектным промптом. Оцените.
4. Покрутите силы. Перегенерируйте 9. Зафиксируйте, когда довольны.
5. Сохраните как T2I-воркфлоу `05-keystill-track[A|B|C].json`.

На стороне LTX-2.3: загрузите T2I-выход как I2V-вход, прогоните
keyframe-воркфлоу из модуля 06. Если стилевая LoRA LTX-2.3
совместима, добавьте её на 0.8 и сравните.

## Упражнение B — Сетка силы 3×3

Возьмите две LoRA, которые правдоподобно сочетаются (например,
style-якорь + detail-акцент). Сгенерируйте сетку 3×3:

```text
                 Accent 0.2    Accent 0.4    Accent 0.6
Anchor 0.5         [A5/a2]       [A5/a4]       [A5/a6]
Anchor 0.7         [A7/a2]       [A7/a4]       [A7/a6]
Anchor 0.9         [A9/a2]       [A9/a4]       [A9/a6]
```

Все остальные настройки идентичны (сид, промпт, шаги). Приколите
сетку на стену/десктоп. Ваш глаз сразу выдернет sweet spot. Это 9
генераций — 15 минут на приличной карте — и сэкономит часы
бесцельного тюнинга.

Повторяйте для каждой новой пары LoRA. Складывайте карточки стеков в
папку `stack_cards/` с thumbnail-сеткой рядом.

## Упражнение C — Симуляция конфликта

Намеренно возьмите две LoRA, которые не согласуются (например,
мягкая акварель + жёсткий cel-shaded стиль). Стекните на 0.7/0.7.
Сгенерируйте. Наблюдайте конкретную кашу.

Затем примените playbook разрешения конфликта:

1. Снизьте «менее важную для этого шота» до 0.3.
2. Если всё ещё мутно, поставьте её в bypass и перегенерируйте с
   одной.
3. Опционально: сгенерируйте два вывода с каждой LoRA отдельно и
   композьте нужные регионы из каждого.

Смысл упражнения — натренировать ощущение «LoRA дерутся». Как только
вы начинаете это видеть, перестаёте генерировать 40 изображений в
погоне за проблемой, которая на самом деле конфликт стека.

## Что сохранить перед тем, как идти дальше

- По одному T2I-воркфлоу на каждую дорожку, которую собираетесь
  исследовать.
- «Stack card» для каждого: какие LoRA, какие силы, какие триггеры,
  один репрезентативный вывод и thumbnail 3×3-сетки.
- Заметки о том, какие LoRA *не прошли* в ваш набор и почему. Это
  дорого даваемая интуиция; записывайте.
- «Бюджет-лог» — сумма трат LoRA, использованная на keeper'ах. Ваши
  будущие стеки стартуют от этих чисел.

Продолжайте с [09-advanced-workflow-patterns.md](09-advanced-workflow-patterns.md).
