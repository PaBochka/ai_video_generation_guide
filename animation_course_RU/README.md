# Курс по AI-анимации — создание смотрибельных анимационных короткометражек в ComfyUI

Практический, прикладной курс, который проведёт вас от «у меня есть рабочий workflow LTX-2.3» до «я умею делать 30-секундные и многоминутные анимационные короткометражки в выбранном стиле, с консистентными персонажами и связной историей».

Курс честен относительно нынешнего состояния технологии: в 2026 году AI-инструменты для видео генерируют клипы по ~5–10 секунд за раз, теряют идентичность персонажей между склейками и редко с первого раза попадают в конкретный стилизованный look. **Весь курс построен с учётом этих ограничений**, а не в попытке делать вид, что их нет.

---

## Предварительные требования

К этому моменту вы уже должны уверенно работать с:

- Основами ComfyUI — загрузка workflow, очередь, поиск нод, установка кастомных нод-паков.
- [**ComfyUI Video Workflow Guide**](../ComfyUI_Video_Workflow_Guide.md) из этого репозитория, особенно:
  - **§9** — базовый пайплайн LTX-2.3 (двухэтапный T2V / I2V со звуком).
  - **§10** — стекирование LoRA, LoRA управления камерой, мультикейфреймовые якоря, зацикливающий сэмплер.
  - **§11** — промптинг Gemma для LTX-2.3 (текстовый энкодер — не CLIP, грамматика другая).

Если у вас ещё нет рабочего одноразового workflow LTX-2.3, сначала соберите его по §9. Рецепты курса предполагают этот пайплайн в качестве движка.

---

## Что вы сможете делать в конце

- Планировать последовательность: идея → сценарий → раскадровка → список кадров → промпты.
- Консистентно воспроизводить целевой стиль на множестве кадров (например, **Spider-Man: The Animated Series (1994)**, **80s OVA anime**, **современное cel-shaded аниме**, **субботнее ограниченно-анимированное ТВ**).
- Держать персонажа (лицо, костюм, цветовая палитра) узнаваемым между склейками.
- Генерировать покадровый материал с правильным типом кадра, движением камеры и моушеном.
- Собирать кадры в сцену со звуком, музыкой, диалогами и цветовой непрерывностью.
- Экспортировать в нужном формате для YouTube, TikTok или локального просмотра.

## Предлагаемый учебный путь

**Неделя 1 — Основы**
- Прочитайте [01_animation_fundamentals.md](01_animation_fundamentals.md). Посмотрите одну серию в вашем целевом стиле с блокнотом. Запишите: количество кадров, среднюю длину кадра, движения камеры, повторяющуюся цветовую палитру.
- **Упражнение**: возьмите 30-секундную сцену из существующей анимации. Разбейте её на список кадров с таймкодами.

**Неделя 2 — Стиль**
- Прочитайте [02_style_replication.md](02_style_replication.md). Протестируйте каркасы промптов для выбранного стиля на трёх разных сценах (интерьер, экстерьер, крупный план персонажа).
- **Упражнение**: произведите один 5-секундный клип в целевом стиле с приемлемым качеством. Итеративно шлифуйте каркас промпта, пока не залочится.

**Неделя 3 — Персонажи**
- Прочитайте [03_character_consistency.md](03_character_consistency.md). Соберите **character sheet** для одного персонажа (фронт, 3/4, профиль, выражения лица).
- **Упражнение**: сгенерируйте того же персонажа в трёх разных кадрах (CU, MS, WS) и сравните консистентность. Попробуйте маршрут IP-Adapter, затем маршрут LoRA, если захотите лучшего результата.

**Неделя 4 — Производство кадров**
- Прочитайте [04_shot_workflows.md](04_shot_workflows.md). Подберите правильный рецепт под каждый кадр из вашего Week 1 shot list.
- **Упражнение**: произведите все кадры из списка. Пока не заморачивайтесь о звуке.

**Неделя 5 — Пост-продакшн**
- Прочитайте [05_post_production.md](05_post_production.md). Склейте кадры в DaVinci Resolve (бесплатная версия). Добавьте музыку, звуковые эффекты, базовую цветокоррекцию.
- **Упражнение**: экспортируйте финальную 30-секундную короткометражку. Посмотрите её три раза. Отметьте, что ломает погружение. Перерендерите эти кадры.

**Неделя 6+ — Масштабирование**
- Примените тот же пайплайн к короткометражке на 2–5 минут.
- Обучите собственную character LoRA (кратко затронуто в модуле 3).
- Соберите рекуррентную style LoRA, чтобы не перенастраивать каркас промпта каждый раз.

## Итоговый проект (capstone)

Выберите одно:

1. **30-секундная экшн-сцена** — Spider-Man-94 раскачивается на паутине над крышами. Два персонажа, три кадра, один музыкальный cue.
2. **30-секундный аниме-колд-опен** — атмосферный establishing-кадр, появление персонажа, жёсткая склейка в титул. Один персонаж, три кадра, амбиентный звук.
3. **Зацикленный клип для музыкального видео** — 8-секундная зацикливаемая сцена, синхронизированная под музыкальный бэд. Один персонаж, один кадр, looping sampler.

Какой бы вы ни выбрали, курс даёт все нужные техники. Capstone — это то место, где вы собираете их end-to-end.

## Глоссарий

| Термин | Значение |
|--------|----------|
| **Shot (кадр, план)** | Один непрерывный дубль — одна камера, без склеек. Атомарная единица монтажа. |
| **Scene (сцена)** | Серия кадров в одном месте / времени. |
| **Sequence (последовательность)** | Серия сцен, образующих связный сюжетный бит. |
| **Cut (склейка)** | Переход между двумя кадрами. |
| **Establishing shot** | Широкий кадр, показывающий локацию и устанавливающий сцену. |
| **Close-up (CU)** | Плотная рамка на лицо или объект. |
| **Medium shot (MS)** | Персонаж от пояса. |
| **Two-shot** | В кадре оба персонажа. |
| **Over-the-shoulder (OTS)** | Кадр из-за плеча одного персонажа, смотрящего на другого. |
| **Insert** | Короткий кадр детали (часы, рука, объект). |
| **Beat** | Один драматический момент внутри сцены. |
| **Storyboard (раскадровка)** | Последовательность рисунков (один на кадр) с камерой, действием и диалогами. |
| **Shot list (список кадров)** | Таблица/документ с одной строкой на кадр: номер, тип, длина, камера, действие, диалог. |
| **Cel-shaded** | Look, определяемый плоскими цветовыми зонами и жирной tushевой обводкой (мультфильмы, аниме). |
| **Limited animation** | Стиль с малым количеством рисунков в секунду; использует held cels, смэр-фреймы, панорамирование. |
| **Held cel** | Один кадр, удерживаемый много видеокадров (губы двигаются, тело — нет). |
| **Smear frame** | Один искажённый кадр, подразумевающий быстрое движение. |
| **Key frame / anchor frame** | Кадр, в который AI обязан попасть точно. В LTX-2.3: вход `LTXVAddGuide`. |
| **In-between** | Кадр между двумя ключевыми. AI генерирует их неявно. |
| **LoRA** | Маленькие адаптерные веса, смещающие модель в сторону стиля или персонажа. См. workflow guide §10. |
| **IP-Adapter** | Позволяет обусловливать генерацию референсным изображением без обучения LoRA. |
| **FLF2V** | First-Last Frame to Video. Форма мультикейфреймового I2V. |
| **Looping sampler** | Сэмплер LTX-2.3, генерирующий длинные клипы перекрывающимися тайлами. См. workflow guide §10.5. |

## Честные границы возможного на 2026 год

- **Длина клипа за одну генерацию**: ~10 с @ 25 fps без падения качества (LTX-2.3 с looping sampler).
- **Консистентность персонажа между склейками**: трудно. Лучше всего с обученной character LoRA. IP-Adapter + референс-фрейм — на втором месте.
- **Lip-sync под диалог**: требует пост-обработки (Wav2Lip, SadTalker). В LTX-2.3 нативно отсутствует.
- **Хореография экшена**: простые действия — ок; сложные многоперсонажные драки — очень трудно. Планируйте с учётом этого.
- **Специфическая стилизация** (рисованное аниме, cel-shaded): требует style LoRA или пост-обработки. Чистыми промптами выжимается ~60%.
- **Несколько рекуррентных персонажей в одном кадре**: труднее, чем один персонаж. Часто проще генерировать соло-кадры и композировать.

Эти границы будут сдвигаться. Перечитайте этот раздел через 6 месяцев и через год — релизы моделей сдвинут то, что считается разумным.
